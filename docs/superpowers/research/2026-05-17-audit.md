## 7. Strengths shipped — DO NOT reinvent in research

1. **Event-sourced ledger with HLC + LWW + cursor + outbox + sync_log.** SUM-of-deltas semantics for inventory, append-only audit trail, idempotent UPSERT-by-event_uuid replay, deterministic conflict resolution under ±24h clock skew (100-case Kotest property proof at `LwwClockSkewPropertyTest`), (timestamp, eventUuid) half-open pull-since cursor (20-case property proof at `PullCursorPropertyTest`), per-trigger sync_log fan-in observability.
2. **Materialized `pantry_snapshot` with trigger-replacement-via-EventStore.** O(distinct SKUs) reads instead of O(history). Benchmarked ~4 ms at 100k events / 200 SKUs (target <10 ms). `PantryBenchmarkTest` proves.
3. **Ack-vs-flip atomicity.** `AckVsFlipChaosTest` simulates kill between server 200 ACK and local `markSynced`; replay produces no double-apply because server-side UPSERT-by-UUID + client snapshot uses trigger-equivalent same-tx delta.
4. **Schema-parity gate** Flyway PG ↔ SQLDelight SQLite via Testcontainers `pgvector/pgvector:pg16`. Type-alias allow-list (UUID↔TEXT, TIMESTAMPTZ↔INTEGER, JSONB↔TEXT, BOOLEAN↔INTEGER, BIGSERIAL↔INTEGER PRIMARY KEY AUTOINCREMENT, VECTOR↔BLOB). 1/1 passing.
5. **WAL pragmas + Doze chaos test.** `journal_mode=WAL; synchronous=NORMAL; busy_timeout=5000; cache_size=-64000; foreign_keys=ON; wal_autocheckpoint=1000`. Robolectric `WalDozeChaosTest` (data persists across simulated kill+reopen+checkpoint) + `WalCheckpointTest` (wal-file size shrinks after `forceTruncatingCheckpoint`).
6. **CI workflow with Docker-backed tests.** `.github/workflows/ci.yml` on `branches: ['**']` with `timeout-minutes: 30`. JDK 21 setup-java + Gradle cache + ktlint + detekt + `:shared:desktopTest` + `:shared:testDebugUnitTest` + `:server:test` + assembly.
7. **Sealed `OutboxError` + transport vs permanent classification.** Transient transport (5xx / IOException / SocketTimeoutException) does NOT increment dead-letter ceiling; only 4xx `PushRejected` or persistent server errors promote.

9. **`CancellationException` re-throw discipline** across `OutboxDrainWorker`, `WebSocketListener`, `NtfyClient` — `try { ... } catch (e: CancellationException) { throw e } catch (e: Throwable) { ... }` pattern (NOT `runCatching` which swallows cancellation).
